# 用户模块

## 需求
- 用户注册： 用户可以通过输入账号、密码、确认密码等信息进行注册
- 用户登录：用户可以通过输入账号、密码进行登录
- 获取当前登录用户：得到当前登录用户的用户信息
- 用户注销：用户可以退出当前登录状态
- 用户权限控制：用户根据不同的角色，有不同的权限，例如普通用户、管理员等
- 【管理员】管理用户：管理员可以对普通用户进行注册、登录、注销、权限控制等操作


## 用户登录流程
- 建立初始会话：前端与服务器建立连接后，服务器会为客户端创建一个初始匿名 Session，并将其状态保存下来，这个 Session 的 ID 会作为唯一标识，返回给前端。
- 登录成功，更新会话信息：当用户在前端输入正确的账号密码并提交到后端验证成功后，后端会更新该用户的 Session 信息，包括用户 ID、角色等。保存到与这个 Session 关联的数据库记录中。同时，服务器会生成一个 Set-Cooke的响应头，提示前端保存该用户的 Session ID 
- 前端保存 Cookie：前端接收到后端的响应后，会将 Set-Cookie 中的 Session ID 保存到本地 Cookie 中，后续请求会携带这个 Cookie。
- 带 Cookie 的后续请求：当前端再次向相同域名的服务器发送请求时，浏览器会自动在请求头中附带之前保存的 Cookie ， 包含 Session ID 。
- 后端验证 Session ID：后端收到请求后，会从 Cookie 中提取 Session ID ，并验证该 Session ID 是否有效。如果有效，后端会根据 Session ID 从数据库中查询用户信息，将用户信息返回给前端。

## 用户权限
传统的权限控制方法是，在每个接口内单独编写权限检查代码：先获取到当前登录用户的信息，然后判断用户的权限是否符合要求。
这种方法最灵活，但是会导致代码重复，增加维护成本。其他开发者无法第一眼就知道接口所需要的权限。

权限校验其实是一个比较通用的业务需求，一般会通过 ** Spring AOP 切面 + 自定义权限校验注解 ** 实现统一的接口拦截和权限校验；如果有特殊的权限校验逻辑，再单独在接口中编写。

## Mybatis Flex 代码在生成器
MyBatis-Flex 代码生成器，可以理解成：帮你自动生成一整套 CRUD 代码的工具，省掉大量手写重复代码。
https://mybatis-flex.com/zh/intro/comparison.html

1️⃣ 它能生成什么
根据数据库表，一键生成：
Entity（实体类）
Mapper（接口）
Mapper.xml（可选）
Service / ServiceImpl
Controller（可选）
👉 基本就是把你平时写的那一套后端分层代码全自动搞出来。

2️⃣ 原理很简单

本质就是： 读取数据库表结构 → 按模板拼代码 → 输出到项目里
